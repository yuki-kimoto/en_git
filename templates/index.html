Git講座 - プロのWeb開発者のソースコード管理

実践Git入門は、プロのWeb開発者の現場で使われているソースコード管理ツールGitを学べる講座です。GitはLinuxサーバー上でWebアプリケーションのプログラムのソースコードやHTML・CSSを管理するために実務で広く活用されているコマンドラインツールです。 
# ソースコードをコミット(ファイル管理情報に追加)する
git commit -m "Fix web api bug"

# ソースコード管理情報をリモートサーバーに転送
git push origin main

# チームの他のメンバーが修正した内容を取り込む
git merge tanaka_web_api_fix

Gitコマンド

  • git init - Gitリポジトリの初期化
  • git log - コミットログの内容を見る
  • git status - コミット準備されているかを確認する
  • git add - ファイルをコミット準備状態にする
  • git commit - コミットする
  • git branch - ブランチの一覧表示・ブランチの作成
  • git checkout - ブランチの変更
  • git diff - 差分の表示
  • git reset - コミットの位置を移動
  • git fetch - リモートリポジトリの情報取得
  • git remote - リモートリポジトリの追加・変更・削除
  • git push - ローカルリポジトリの内容をリモートリポジトリに転送
  • git pull - リモートリポジトリの内容をローカルリポジトリに転送してマージ
  • git tag - タグをつける
  • git revert - 元に戻す変更内容でコミット
  • git merge - マージする
  • git rebase - 複数のコミットを改変してまとめる
